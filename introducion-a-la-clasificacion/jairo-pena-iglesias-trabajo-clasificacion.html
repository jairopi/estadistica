Implementacin de la funcin coste

Vamos ahora a implementar la funcin coste que llamaremos J. En las transparencias, encontramos la expresin para esa funcin coste en el caso de la regresin logstica (recordad que los datos y_i son datos binarios) \[J(\theta)=- \frac 1 n\sum_{i=1}^n \left\{y_i\log(h_\theta(x_{i\bullet}))+(1-y_i)\log(1-h_\theta(x_{i\bullet}))\right\}\]

Empezamos por crear la matriz x.mat de diseo as como el vector y de observaciones para poder usarlos en la definicin de la funcin de coste.

## nada que completar
x.mat <- cbind(1, as.matrix(datos %>% select(-y)))
y <- datos$y
J=function(theta)
  {
  ## Completar aqu para calcular J.tmp. Pagina 38 (Trasparencia 59)
  J.tmp = (-1/length(y)) * sum(y * log(g(x.mat%*%theta)) + (1-y) * log(1-g(x.mat%*%theta)))
  
  ##
  return(J.tmp)
  }

Podemos probar algunos valores de theta para comprobar vuestra implementacin: Para J(c(0,0,0)) debis encontrar 0.6931. Para J(c(-0.5,-0.5,0.5)), debis encontrar 0.3724.

Implementacin de la funcin gradiente del coste

Para poder usar algortmos de bsqueda del mnimo de la funcin de coste, implementaremos tambin la funcin gradiente del coste, cuya expresin es (ver transparencias)

Si usamos la matriz de diseo \(\mathbf{X}\), obtenemos en forma compacta: \[ \nabla J=\frac 1 n \mathbf{X}^T\cdot\left(\mathbf{H}_\theta-y\right),\] donde \(\mathbf{H}\) denota el vector columna: \[\mathbf{H}_\theta=\left( \begin{array}{l} h_\theta(x_{1\bullet})\\ h_\theta(x_{2\bullet})\\ \vdots\\ h_\theta(x_{n\bullet})\\ \end{array}\right)\]

gradJ=function(theta)
  {
  # tenes que calcular el gradiente aqu. Pagina 40 (trasparencia 63)
  gradJ = (1/length(y)) * (t(x.mat) %*% (g(x.mat %*% theta) - y))
  return(gradJ)
  }

Bsqueda del mnimo

Una vez que tenemos implementada la funcin de coste, podemos escribir el cdigo para una bsqueda del mnimo. Usaremos, en lugar de programar nosotros mismos el algoritmo del gradiente, la funcin optim() de R, que implementa mtodos optimizados de bsqueda del mnimo. Especificamos como argumentos la funcin que queremos minimizar (argumento fn), su gradiente (argumento gr) y el mtodo que queremos emplear (argumento method). Usaremos el mtodo BFGS que consiste en una mejora del mtodo del gradiente. Tambin tenemos que especificar valores iniciales de los parmetros

## nada que completar
optim.results=optim(c(0,0,0), fn=J, gr=gradJ, method = "BFGS")

Guardamos los resultados del algoritmo de minimizacin en un vector que llamamos thetaopt.

Cul es la ecuacin de la frontera de decisin?

Cul es la forma de la frontera de decisin? Se trata de un circulo.

En la grfica que tenamos del conjunto, vamos a aadir la frontera de decisin.

## Nada que completar aqu
m1 <- - thetaopt[2] / (2 * thetaopt[4])
m2 <- - thetaopt[3] / (2 * thetaopt[5])
R2 <- thetaopt[2]^2 / (4 * thetaopt[4]) +
  thetaopt[3]^2 / (4 * thetaopt[5]) -
  thetaopt[1]
r1 <- sqrt(R2 / thetaopt[4])
r2 <- sqrt(R2 / thetaopt[5])
x1e <- seq(m1 - r1, m1 + r1, length.out = 100)
deltay1e <- sqrt(r2^2 * (1-(x1e-m1)^2 / r1^2))

elipse <- data.frame(x1 = c(x1e, rev(x1e)),
                     x2 = c(m2 + deltay1e,
                            m2 - deltay1e))
datos %>%
  ggplot(aes(x1, x2)) + geom_point(aes(colour = factor(y))) +
           geom_path(data = elipse)
